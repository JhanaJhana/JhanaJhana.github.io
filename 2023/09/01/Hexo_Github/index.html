如何用 Hexo 和 Github 搭建個人網誌

Posted on 2023-09-01 Edited on 2024-03-22 In Website Disqus:

放暑假看了很多分享NLP的blog之後想想，咦！要不我也來搭建一個人網誌吧！！

  • 前言： 雖然已經有 HackMD，但在 HackMD 上面主要寫自己上線上課程還有線下在校修課的Learning Notes。因此決定要一個自己的網站，預計可以作為分享閱讀心得或小專案分享使用。

第一篇就決定是：如何用Hexo和Github搭建網站！

Q: 為什麼選Hexo呢？

核心原因是因為想要一個乾淨簡潔的網誌，在看了網路許多分享，發現我喜歡的風格都是使用 Hexo 搭建的！
所以進一步了解，又發現了以下的優點：

  1. 可以使用 Markdown 語法！程式碼可以很簡潔的顯示。
  2. 方便的外掛系統。
  3. 有豐富簡潔、內容清晰排版的主題可以選擇。
  4. 可以放在 Github 的儲存庫。
  5. 而且免費。
  6. 想要嘗試看看大名鼎鼎的 Node.js 。

所以首先可以先去了解了一下 Node.js

  • Node.js 使 Javascript 從前端延伸到後端，讓電腦可以用 Command line 的方式執行 JavaScript

  • Hexo

Deploy on Github

Hexo 部署的方式有很多種，這裡選擇了常見、方便的方法：Github （可以存放靜態網頁，用 .github.io 的子網域）。

Create a repository

首先得創建一個 Github 帳戶 ，並創建新的 Repository(儲存庫)。

Repository 要命名為 <github-username>.github.io ，並且設定為 public。

首先得先部署外掛

回到_config.yml檔案中的 Deployment，把 repository 後加上剛剛在 Github 中創建的 Repository網址。（格式：https://github.com/用戶名/用戶名.github.io.git）

再回到終端機上，就可以把內容 deploy 到 Github 上面囉！

更多 Deployment 的相關資訊

  • Note: GitHub 不支援密碼驗證上傳解決方案

    其實錯誤提示寫得非常潛顯易懂，所以也不用太緊張啦～只要 Github personal access token 就可以了。

Bonus: 引入外掛系統

開始認真編輯網頁之後就發現：
沒有辦法互動耶！不像是HackMD有可以留言的地方，這裡找不到可以留言的地方，這樣就無法和來我網站的人互動。

想來想去，就上網看看大家都怎麼解決這件事，發現許多人都有一樣的想法。
因此最後我又引入外掛的留言程式，讓每一篇文章下方都可以有留言功能。
評比了多個程式之後，覺得 Disqus 長得最順眼簡潔，符合我的網誌期望的風格（而且免費！）。
https://www.techcoke.com/2013/11/blogger-quick-installation-disqus-message-board.html

首先，一樣得先創一個新帳號 （Disqus的官網）並命名 shortname。

接著回到 _config.yml 檔案中，找到 disqus，把 enable 改成true。

此時，每一篇文章下方都會有留言功能啦！

  • Note: 如果只是單純這樣改，會發現Menu中的頁面（例如：About）也都會出現留言功能。

    因此，需找到其他不需要留言功能的頁面 index.md 檔案中的 Front-matter，修改 comment 設定。
